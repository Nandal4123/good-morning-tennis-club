## 🎯 요구사항 1: 피드백 수집 시스템

### 구현 내용
사용자가 앱 내에서 직접 피드백을 남길 수 있는 시스템

### 필요한 기능
1. **피드백 폼**
   - 사용 빈도 (일일/주간/월간)
   - 가장 유용한 기능 선택
   - 개선이 필요한 부분
   - 유료 사용 의향 (예/아니오)
   - 적정 가격대 선택
   - 추가 의견 (텍스트)

2. **피드백 저장**
   - 데이터베이스에 저장
   - 클럽별/사용자별 구분
   - 날짜 기록

3. **피드백 조회 (관리자용)**
   - 모든 피드백 목록
   - 클럽별 필터링
   - 통계 요약

### 요구 방식
```
"피드백 수집 시스템을 만들어줘. 
- 앱 내에 '피드백 남기기' 버튼 추가
- 피드백 폼에 다음 항목 포함:
  * 사용 빈도 (일일/주간/월간)
  * 가장 유용한 기능 (다중 선택)
  * 개선이 필요한 부분 (텍스트)
  * 유료 사용 의향 (예/아니오)
  * 적정 가격대 (2만원/3만원/5만원/10만원)
  * 추가 의견
- 관리자 대시보드에 피드백 목록 표시
- 클럽별 통계 표시"
```

## 🎯 요구사항 2: 사용량 추적 시스템

### 구현 내용
각 클럽의 사용량을 자동으로 추적하는 시스템

### 필요한 기능
1. **사용량 기록**
   - 일일 활성 사용자 수
   - 경기 등록 수
   - 출석 체크 수
   - 기능별 사용 횟수

2. **사용량 저장**
   - 데이터베이스에 일별 기록
   - 클럽별 구분

3. **사용량 조회 API**
   - 클럽별 사용량 조회
   - 기간별 통계
   - 전체 통계

### 요구 방식
```
"사용량 추적 시스템을 만들어줘.
- 각 클럽의 일일 사용량을 자동으로 기록:
  * 활성 사용자 수
  * 경기 등록 수
  * 출석 체크 수
  * 각 페이지 방문 수
- 데이터베이스에 일별로 저장
- API 엔드포인트 추가: GET /api/analytics/usage
- 관리자 대시보드에 사용량 통계 표시"
```

## 🎯 요구사항 3: 베타 테스터 관리 시스템

### 구현 내용
베타 테스트 클럽을 관리하는 시스템

### 필요한 기능
1. **베타 테스터 등록**
   - 클럽 정보 입력
   - 담당자 정보
   - 시작일/종료일
   - 상태 (활성/종료)

2. **베타 테스터 목록**
   - 등록된 클럽 목록
   - 상태별 필터링
   - 연락처 정보

3. **베타 테스터 대시보드**
   - 클럽별 사용량 요약
   - 피드백 요약
   - 상태 관리

### 요구 방식
```
"베타 테스터 관리 시스템을 만들어줘.
- 데이터베이스에 BetaTester 모델 추가:
  * 클럽 이름
  * 담당자 이름/이메일
  * 시작일/종료일
  * 상태 (ACTIVE/ENDED)
- 관리자 페이지에 베타 테스터 목록 표시
- 클럽별 사용량 및 피드백 요약 표시"
```

   - 피드백 제출 수
   - 유료 전환 의향 비율
   - 적정 가격대 분포
   - 개선 요구사항 요약

### 요구 방식
```
"관리자용 분석 대시보드를 만들어줘.
- 전체 통계 카드:
  * 총 클럽 수
  * 총 사용자 수
  * 총 경기 수
  * 총 출석 수
- 사용량 차트:
  * 일일/주간/월간 사용량 그래프
  * 클럽별 사용량 비교
- 피드백 통계:
  * 피드백 제출 수
  * 유료 전환 의향 비율
  * 적정 가격대 분포
- Dashboard 페이지에 추가"
```

```
- 베타 테스터용 빠른 시작 가이드 추가
```

## 📝 단계별 구현 요구사항

### Phase 1: 기본 인프라 (1주)

#### 요구 1-1: 피드백 데이터베이스 모델
```
"피드백을 저장할 데이터베이스 모델을 만들어줘.
- Feedback 모델에 다음 필드 추가:
  * clubName (클럽 이름)
  * userId (사용자 ID)
  * usageFrequency (사용 빈도)
  * usefulFeatures (유용한 기능, JSON 배열)
  * improvements (개선사항, 텍스트)
  * willingToPay (유료 의향, Boolean)
  * priceRange (적정 가격대)
  * comments (추가 의견)
  * createdAt (생성일)
- Prisma 스키마에 추가하고 마이그레이션 실행"
```

#### 요구 1-2: 사용량 추적 데이터베이스 모델
```
"사용량 추적을 위한 데이터베이스 모델을 만들어줘.
- UsageLog 모델 추가:
  * clubName (클럽 이름)
  * date (날짜)
  * activeUsers (활성 사용자 수)
  * matchesCreated (경기 등록 수)
  * attendances (출석 수)
  * pageViews (페이지 뷰, JSON)
  * createdAt (생성일)
- Prisma 스키마에 추가하고 마이그레이션 실행"
```

#### 요구 1-3: 베타 테스터 데이터베이스 모델
```
"베타 테스터 관리를 위한 데이터베이스 모델을 만들어줘.
- BetaTester 모델 추가:
  * clubName (클럽 이름)
  * contactName (담당자 이름)
  * contactEmail (담당자 이메일)
  * startDate (시작일)
  * endDate (종료일)
  * status (ACTIVE/ENDED)
  * notes (메모)
  * createdAt (생성일)
- Prisma 스키마에 추가하고 마이그레이션 실행"
```

### Phase 2: 백엔드 API (1주)

#### 요구 2-1: 피드백 API
```
"피드백 API를 만들어줘.
- POST /api/feedback - 피드백 제출
- GET /api/feedback - 피드백 목록 조회 (관리자용)
- GET /api/feedback/stats - 피드백 통계
- FeedbackController와 FeedbackRoutes 생성"
```

#### 요구 2-2: 사용량 추적 API
```
"사용량 추적 API를 만들어줘.
- POST /api/analytics/log - 사용량 기록 (자동 호출)
- GET /api/analytics/usage - 사용량 조회
- GET /api/analytics/stats - 전체 통계
- AnalyticsController와 AnalyticsRoutes 생성
- 미들웨어로 자동 사용량 기록"
```

#### 요구 2-3: 베타 테스터 API
```
"베타 테스터 관리 API를 만들어줘.
- POST /api/beta-testers - 베타 테스터 등록
- GET /api/beta-testers - 베타 테스터 목록
- PUT /api/beta-testers/:id - 베타 테스터 수정
- BetaTesterController와 BetaTesterRoutes 생성"
```
